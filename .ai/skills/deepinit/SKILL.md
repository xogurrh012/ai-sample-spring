# Deep Init

코드베이스를 탐색하여 각 디렉토리에 AI가 이해할 수 있는 계층형 AI-CONTEXT.md 파일을 생성하는 스킬이다.

## 언제 사용하는가

- 새 프로젝트에 AI 가이드 문서를 처음 만들 때
- 코드베이스가 커져서 구조 문서화가 필요할 때
- AI 에이전트가 프로젝트를 빠르게 이해해야 할 때

## AI-CONTEXT.md 파일 구조

각 AI-CONTEXT.md에는 다음이 포함된다:

```markdown
<!-- Parent: ../AI-CONTEXT.md -->

# [디렉토리명]

## 목적

[이 디렉토리의 역할 한줄 설명]

## 주요 파일

| 파일명     | 설명   |
| ---------- | ------ |
| file1.java | [역할] |
| file2.java | [역할] |

## 하위 디렉토리

- `sub1/` - [설명]
- `sub2/` - [설명]

## AI 작업 지침

- [이 디렉토리에서 작업할 때 주의할 점]
- [사용하는 패턴/컨벤션]

## 테스트

- [테스트 실행 방법]

## 의존성

- 내부: [참조하는 다른 모듈]
- 외부: [사용하는 라이브러리]
```

## 동작

### Step 1: 디렉토리 구조 매핑

프로젝트의 전체 디렉토리 구조를 파악한다.

### Step 2: 작업 계획

깊이(depth) 레벨별로 작업 순서를 정한다:

- 부모 디렉토리를 자식보다 먼저 처리
- 같은 레벨은 병렬 처리 가능

### Step 3: 레벨별 문서 생성

각 디렉토리를 순회하며 AI-CONTEXT.md를 생성한다:

- 빈 디렉토리는 건너뜀 (하위 디렉토리가 있는 경우 제외)
- 기존 AI-CONTEXT.md가 있으면 비교 후 업데이트
- `<!-- MANUAL -->` 태그가 있는 섹션은 보존

### Step 4: 계층 검증

- 모든 Parent 참조가 유효한 경로를 가리키는지 확인
- 파일 설명이 정확한지 확인
- 누락된 디렉토리가 없는지 확인

## 핵심 원칙

- **정확성 우선**: 파일 설명이 실제 코드와 일치해야 한다
- **계층 연결**: Parent 참조로 트리 탐색이 가능해야 한다
- **실용적 지침**: 일반적 보일러플레이트가 아닌 실제 유용한 AI 작업 지침
- **수동 섹션 보존**: `<!-- MANUAL -->` 태그 내용은 업데이트 시에도 유지

## 주의사항

- 대규모 프로젝트에서는 시간이 걸릴 수 있다
- 같은 깊이 레벨은 병렬, 다른 깊이는 순차 처리
- node_modules, .git, build 등 무시할 디렉토리를 먼저 식별한다
